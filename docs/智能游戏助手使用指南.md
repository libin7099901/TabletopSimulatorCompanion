# 🎲 桌游伴侣 v4.0 智能游戏助手使用指南

## 📖 概述

桌游伴侣 v4.0 是一个革命性的 Tabletop Simulator 智能助手，它不仅能回答游戏规则问题，更能实时感知游戏状态，提供上下文相关的策略建议。

## 🎯 核心特性

### 1. 智能上下文感知
- **自动游戏状态收集**：实时获取桌面对象、玩家信息、游戏状态
- **对象智能分类**：自动识别卡牌、骰子、标记等游戏元素
- **玩家状态跟踪**：监控玩家手牌数量、持有物品等信息

### 2. 规则文档管理
- **灵活规则加载**：支持文本格式的游戏规则
- **智能规则应用**：将规则与当前游戏状态结合分析
- **持久化存储**：规则文档自动保存，下次启动时恢复

### 3. 交互方式多样化
- **文本命令**：传统的聊天命令方式
- **右键菜单**：直接在游戏对象上快速查询
- **UI界面**：直观的图形界面控制

## 🚀 快速设置指南

### 步骤1：启动Ollama服务
```bash
# 启动Ollama服务
ollama serve

# 拉取推荐模型
ollama pull gemma3:12b
```

### 步骤2：在TTS中加载脚本
1. 在 Tabletop Simulator 中打开任意游戏
2. 按 `Alt+Tab` 或点击 "Scripting" 按钮
3. 将 `TabletopCompanion.ttslua` 内容粘贴到全局脚本中
4. 点击 "Save & Play"

### 步骤3：配置AI服务
```
tc config ollama http://localhost:11434 gemma3:12b
```

### 步骤4：测试连接
```
tc test
```

看到 "✅ Ollama测试成功！" 表示配置正确。

## 📚 规则文档管理

### 加载游戏规则
```
tc rules load 这是一个策略游戏。玩家轮流在网格上放置建筑物。住宅建筑每个得1分，商业建筑需要相邻住宅才能激活，每个激活的商业建筑得2分。工业建筑会降低相邻住宅的分数。游戏在所有格子被填满时结束，分数最高者获胜。
```

### 查看规则状态
```
tc rules status
```

### 清除规则
```
tc rules clear
```

## 🎮 游戏上下文管理

### 查看当前游戏状态
```
tc context show
```

示例输出：
```
当前游戏: 策略建筑游戏
玩家数量: 3
就座玩家: Red, Blue, White
桌面对象总数: 45
- 卡牌: 12张
- 骰子: 2个
- 标记: 31个
- 其他: 0个
```

### 手动刷新上下文
```
tc context refresh
```

### 控制自动上下文收集
```
tc context auto on    # 启用自动收集
tc context auto off   # 禁用自动收集
```

### 控制收集内容
```
tc context objects on/off  # 包含/排除对象信息
tc context players on/off  # 包含/排除玩家信息
```

## 💡 最佳实践

### 1. 规则文档编写建议
```
# 好的规则文档示例
tc rules load 游戏名称：区域控制大师
游戏目标：通过建筑放置控制更多区域获得胜利
基础规则：
1. 每回合每位玩家可以放置一个建筑
2. 建筑类型：住宅(1分)、商业(2分，需要相邻住宅)、工业(-1分给相邻住宅)
3. 特殊规则：商业建筑必须与至少一个住宅相邻才能得分
4. 胜利条件：所有格子填满时分数最高者获胜
```

### 2. 有效查询技巧
```
# 具体而非抽象的问题
✅ @tc 我在左上角有2个住宅，右边有商业建筑的空地，应该放置商业建筑吗？
❌ @tc 我应该怎么玩？

# 包含当前状态的问题
✅ @tc 现在红色玩家领先5分，我应该采取什么策略追赶？
❌ @tc 如何获胜？

# 明确的规则询问
✅ @tc 商业建筑的得分条件是什么？
❌ @tc 建筑有什么用？
```

### 3. 上下文管理优化
```
# 在复杂游戏中可能需要选择性收集
tc context objects off  # 如果对象太多影响性能
tc context players on   # 保持玩家信息收集

# 在快节奏游戏中可能需要手动控制
tc context auto off     # 禁用自动收集
tc context refresh      # 需要时手动刷新
```

## 🔍 故障排除

### 常见问题

**1. AI无响应**
- 检查：`tc test` 确认连接正常
- 确认Ollama服务运行：命令行运行 `ollama serve`
- 检查模型是否下载：`ollama list`

**2. 上下文信息不准确**
- 手动刷新：`tc context refresh`
- 检查自动收集状态：`tc status`
- 重新启动脚本

**3. 规则文档丢失**
- 规则文档会自动保存，检查：`tc rules status`
- 如果丢失需要重新加载：`tc rules load ...`

**4. UI显示异常**
- 尝试最小化再恢复UI
- 重新加载脚本
- 检查TTS版本兼容性

### 性能优化建议

**对象数量过多时**
```
tc context objects off  # 禁用对象收集
```

**响应速度慢时**
```
tc context auto off     # 禁用自动收集
# 需要时手动刷新
tc context refresh
```

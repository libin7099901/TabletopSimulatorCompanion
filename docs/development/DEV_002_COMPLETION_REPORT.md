## 🏗️ 架构设计

### 模块依赖关系
```
StorageManager (核心存储)
├── ConfigManager (配置管理)
├── CacheManager (缓存管理) 
└── DataCompressor (数据压缩)
```

### 数据流设计
1. **配置数据**: DefaultConfigs → ConfigManager → StorageManager
2. **缓存数据**: CacheManager → StorageManager (持久化)
3. **压缩数据**: 所有大数据 → DataCompressor → StorageManager

### 存储层级
1. **内存层**: 高速访问缓存
2. **script_state层**: TTS持久化存储  
3. **用户输入层**: 敏感数据每次输入

## 🔧 技术实现

### 关键技术突破

1. **简单RLE压缩算法**: 
   - 自研实现，适合JSON文本压缩
   - 连续字符压缩率可达60%
   - 对TTS存储限制优化良好

2. **LRU缓存实现**:
   - 双向链表 + 哈希表优化
   - O(1)时间复杂度访问和更新
   - 内存高效的淘汰策略

3. **分层配置系统**:
   - 运行时 → 用户 → 默认的优先级机制
   - 完整的配置验证和类型检查
   - 模块化配置模式定义

4. **事件驱动架构**:

   - 异步事件处理支持

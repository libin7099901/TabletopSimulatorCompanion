# 桌游伴侣 (Tabletop Companion) 修复报告

## 发现的问题

在检查 `TabletopCompanion.ttslua` 文件时，发现了以下问题：

1. **主要语法错误**：在约1260行处，存在Lua语法错误，错误消息为 `unexpected symbol near 'else'`。
   
   问题代码:
   ```lua
   else
       safe_broadcast("🤖 " .. data.response, nil, player_obj) -- Target response
   else  -- <-- 这里是错误：在else分支内直接出现了另一个else语句
       safe_broadcast("❌ 无法解析Ollama回复。原始响应见日志。", {1,0.5,0}, player_obj)
       log_message("原始Ollama响应: " .. request.text, LOG_LEVEL.ERROR)
   end
   ```

2. **变量名错误**：在处理规则命令的函数中(约1583行)，使用了不存在的变量`subcommand`而不是正确的`sub_command`，导致错误 `attempt to index a nil value`。
   
   问题代码:
   ```lua
   elseif subcommand == "status" then -- Added tc rules status
       -- ...代码内容...
   ```

3. **缺少函数定义**：代码中使用了`log_message`函数，但该函数未在文件开头定义，只在其他地方定义，可能导致某些上下文中无法使用。

4. **缺少必要的依赖**：
   - 需要安装并运行本地HTTP服务器 (`local_rules_server.py`) 才能正常使用规则加载功能
   - 需要配置Ollama或其他LLM服务才能使用AI功能

## 修复措施

1. **语法错误修复**：
   - 修复了嵌套条件语句的语法错误，将无效的双else结构改为正确的if-else嵌套结构
   
   修复后的代码:
   ```lua
   else
       if data and data.response then
           safe_broadcast("🤖 " .. data.response, nil, player_obj) -- Target response
       else
           safe_broadcast("❌ 无法解析Ollama回复。原始响应见日志。", {1,0.5,0}, player_obj)
           log_message("原始Ollama响应: " .. request.text, LOG_LEVEL.ERROR)
       end
   end
   ```

2. **变量名错误修复**：
   - 修正了变量名拼写错误，将`subcommand`改为`sub_command`
   
   修复后的代码:
   ```lua
   elseif sub_command == "status" then -- Added tc rules status
       -- ...代码内容...
   ```

3. **增加函数定义**：
   - 在文件开头添加了`log_message`函数定义和`LOG_LEVEL`常量，确保这些基础函数和常量在使用前已定义
   
   添加的代码:
   ```lua
   -- 确保先定义日志级别
   local LOG_LEVEL = {
       DEBUG = "DEBUG",
       INFO = "INFO",
       WARNING = "WARNING",
       ERROR = "ERROR"
   }
   
   -- 定义日志函数
   local function log_message(msg, level)
       level = level or LOG_LEVEL.INFO
       print("[桌游伴侣 | " .. level .. "] " .. tostring(msg))
   end
   ```

4. **创建测试工具**：
   - 创建了 `test_script.lua` 用于测试简单的修复代码
   - 创建了 `validate_tts_script.lua` 用于验证全部脚本语法
   - 创建了 `fixed_code_test.lua` 专门测试修复后的关键代码段

## 运行桌游伴侣的正确步骤

1. **准备环境**:
   - 确保已安装Python 3.7+
   - 安装必要的Python库: `pip install Flask Flask-CORS requests`

2. **启动本地HTTP服务器**:
   ```bash
   python local_rules_server.py
   ```
   服务器将在 http://localhost:5678 运行

3. **在Tabletop Simulator中加载模块**:
   - 加载一个桌游
   - 加载桌游伴侣模块 (确保使用修复后的版本)

4. **配置LLM服务**:
   在TTS聊天框中输入:
   ```
   tc config ollama http://localhost:11434 gemma3:12b
   ```
   (请根据您的Ollama安装情况调整URL和模型名称)

5. **加载规则文档**:
   在TTS聊天框中输入:
   ```
   tc rules file C:\\path\\to\\your_rules_image.png
   ```
   或直接加载文本规则:
   ```
   tc rules load 这是一个策略游戏。玩家轮流放置建筑...
   ```

6. **使用智能查询**:
   ```
   @tc 基于当前局势，我应该采取什么策略？
   ```

## 技术备注

1. **核心架构**:
   - TTS Mod (`TabletopCompanion.ttslua`): 提供UI和用户交互，通过WebRequest与本地服务器通信
   - 本地服务器 (`local_rules_server.py`): 处理规则文件加载和与Ollama的通信
   - Ollama服务: 提供AI大语言模型能力

2. **代码结构**:
   - 已确认主要语法错误已修复
   - 使用pcall进行错误处理，增强稳定性
   - 采用模块化结构便于维护

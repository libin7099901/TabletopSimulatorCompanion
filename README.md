# 🎲 桌游伴侣 (Tabletop Companion) v4.0

## 智能游戏助手版 - 全面升级

桌游伴侣现已升级为真正的智能游戏助手，具备游戏上下文感知和智能提示词生成能力。

## ✨ 新功能特色

### 🎮 游戏上下文自动获取
- **实时游戏状态分析**：自动收集桌面对象、玩家信息、区域状态
- **智能游戏识别**：自动检测当前游戏名称
- **对象分类统计**：自动分类卡牌、骰子、标记等游戏元素

### 🧠 智能提示词生成
- **上下文感知AI**：将游戏状态自动整合到LLM查询中
- **专业系统提示词**：为AI配置专业桌游助手角色
- **完整上下文构建**：结合规则文档、游戏状态和玩家问题

### 📚 规则文档管理
- **灵活规则加载**：支持文本格式规则文档
- **持久化存储**：规则文档自动保存
- **状态追踪**：实时显示规则文档加载状态

### 🎯 真正的辅助工具功能
- **右键快速查询**：在游戏对象上右键直接询问规则
- **局势分析**：全局右键菜单快速分析当前局势
- **策略建议**：基于游戏状态提供智能策略建议

## 🚀 快速开始

### 1. 基础配置
```
tc config ollama http://localhost:11434 gemma3:12b
```

### 2. 加载规则文档
```
tc rules load 这是一个策略游戏。玩家轮流放置建筑...
```

### 3. 智能查询
```
@tc 基于当前局势，我应该采取什么策略？
```

## 📋 完整命令列表

### 基础命令
- `tc help` - 显示完整帮助
- `tc status` - 显示系统状态
- `tc test` - 测试LLM连接

### 配置命令
- `tc config ollama <URL> <模型>` - 配置Ollama服务

### 上下文命令
- `tc context refresh` - 手动刷新游戏上下文
- `tc context auto on/off` - 启用/禁用自动上下文收集
- `tc context show` - 显示当前游戏状态摘要
- `tc context objects on/off` - 包含/排除对象信息
- `tc context players on/off` - 包含/排除玩家信息

### 规则命令
- `tc rules load <文本>` - 加载规则文档
- `tc rules clear` - 清除当前规则文档
- `tc rules status` - 显示规则文档状态

### 智能查询
- `@tc <问题>` - 向AI提问（自动包含游戏上下文）

## 💡 使用场景示例

### 场景1：策略游戏助手
```bash
# 1. 配置AI服务
tc config ollama http://localhost:11434 gemma3:12b

# 2. 加载游戏规则
tc rules load 这是一个区域控制游戏。玩家通过放置建筑来控制区域，每个区域在游戏结束时根据控制程度给予分数。建筑分为住宅、商业和工业三种类型...

# 3. 开始游戏，系统自动收集上下文
# 4. 智能查询
@tc 我现在有3个住宅和2个商业建筑，应该优先发展哪个区域？

# 5. 或使用右键菜单
# 右键点击桌面 -> "桌游伴侣 - 分析当前局势"
```

### 场景2：规则查询助手
```bash
# 对特定卡牌右键
# 选择"询问规则" -> 自动查询该卡牌的使用规则

# 全局规则查询
@tc 在我的回合中，我可以先移动再攻击吗？
```

## 🔧 技术架构

### 核心模块
- **上下文收集引擎**：基于TTS API实时获取游戏状态
- **智能提示词生成器**：构建包含游戏上下文的完整LLM查询
- **规则文档管理器**：处理规则文档的加载、存储和检索
- **事件监听系统**：自动响应游戏状态变化

### API集成
- **TTS对象API**：getObjects(), Player API, Hands API
- **事件系统**：onObjectDrop, onPlayerChangeColor
- **UI系统**：动态XML界面，右键菜单集成
- **LLM服务**：Ollama API完整支持

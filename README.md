## ⚠️ 重要：运行依赖

本Mod现在依赖一个本地Python HTTP服务器来处理规则文件加载和与Ollama LLM的通信。请务必在启动Tabletop Simulator并使用本Mod前，先运行此本地服务器。

**启动本地服务器步骤 (`local_rules_server.py`)**:
1.  确保您的系统已安装 Python 3.7+。
2.  打开终端或命令行。
3.  安装所需依赖:
    ```bash
    pip install Flask Flask-CORS requests
    ```
4.  导航到 `local_rules_server.py` 文件所在的目录。
5.  运行服务器:
    ```bash
    python local_rules_server.py
    ```
6.  您应该看到服务器启动的日志，通常会提示服务运行在 `http://localhost:5678` 或 `http://0.0.0.0:5678`。**请保持此服务器在您使用Mod期间持续运行。**

## 🚀 快速开始

### 0. 启动本地HTTP中继服务
按照上方 "重要：运行依赖" 部分的说明，启动 `local_rules_server.py`。

### 1. 基础配置 (在TTS聊天框中)
```
tc config ollama http://localhost:11434 gemma3:12b 
```
(注意: Ollama URL 和模型名称应与您本地Ollama服务的配置一致。本地HTTP中继服务默认会尝试连接 `http://localhost:11434` 的Ollama。)

### 2. 加载规则文档 (通过UI或命令)
- **通过UI (推荐)**:
    1. 在Mod的UI界面中，找到"规则文件路径"输入框。
    2. 输入您的规则文件 (图片如 .png, .jpg 或文本如 .txt, .md) 的**完整本地路径** (例如: `C:\\Users\\YourName\\Documents\\MyGameRules.png`)。
    3. 点击"加载本地规则"按钮。
- **通过命令 (备用)**:
    ```
    tc rules file C:\\path\\to\\your_rules_image.png
    ```
    或者加载纯文本规则:
    ```
    tc rules load 这是一个策略游戏。玩家轮流放置建筑...
    ```

### 3. 智能查询
```
@tc 基于当前局势，我应该采取什么策略？
```

## 📋 完整命令列表

### 基础命令
- `tc help` - 显示完整帮助
- `tc status` - 显示系统状态 (包括从本地服务器获取的规则状态)
- `tc test` - 测试LLM连接 (通过本地服务器代理到Ollama)

### 配置命令
- `tc config ollama <URL> <模型>` - 配置Ollama服务参数 (Mod会将这些参数传递给本地服务器的Ollama代理)

### 上下文命令
- `tc context refresh` - 手动刷新游戏上下文
- `tc context auto on/off` - 启用/禁用自动上下文收集
- `tc context show` - 显示当前游戏状态摘要
- `tc context objects on/off` - 包含/排除对象信息
- `tc context players on/off` - 包含/排除玩家信息

### 规则命令 (与本地HTTP服务器交互)
- `tc rules load <文本>` - 加载纯文本规则到本地服务器
- `tc rules file <本地文件路径>` - 请求本地服务器加载指定路径的规则文件（图片或文本）
- `tc rules uri <URL>` - 下载指定URL的文本内容，并加载到本地服务器 (仅支持纯文本URL)
- `tc rules clear` - 请求本地服务器清除当前规则
- `tc rules status` - 显示本地服务器当前的规则加载状态
- `tc rules check` - 让AI确认对当前服务器上加载规则的理解

### 智能查询
- `@tc <问题>` - 向AI提问（自动包含游戏上下文，请求通过本地服务器代理给Ollama）

## 🔧 技术架构

### 核心模块
- **本地HTTP中继服务 (`local_rules_server.py`)**:
    - 使用Python Flask构建。
    - 负责接收Mod发来的文件路径或文本规则，进行文件读取、Base64转码（针对图片）。
    - 在内存中存储当前加载的规则（图片Base64或文本）。
    -作为Mod与Ollama服务之间的代理，将Mod的查询请求（附带已加载的图片规则，如果存在）转发给Ollama。
- **TTS Mod (`TabletopCompanion.ttslua`)**:
    - **上下文收集引擎**：基于TTS API实时获取游戏状态。
    - **智能提示词生成器**：构建LLM查询的文本部分。
    - **规则管理逻辑**：通过WebRequest与本地HTTP服务交互，实现规则加载、清除和状态查询。
    - **事件监听系统**：自动响应游戏状态变化。
    - **UI交互逻辑**: 处理用户在Mod UI上的操作。

### API集成
- **TTS内部API**: getObjects(), Player API, Events API, UI API等。
- **WebRequest (TTS)**: 用于Mod (`.ttslua`) 与本地HTTP服务 (`local_rules_server.py`) 之间的通信。
- **Requests (Python)**: 用于本地HTTP服务代理请求到Ollama服务。
- **LLM服务**: Ollama API (通过本地服务器代理访问)。
